In the second part of Module 3, we discuss the importance of web session management. We recall the original HTTP authentication with limitations arising from its stateless nature. Session management addresses this problem by creating a session — a series of related interactions between a user and the web application or system during a specific period. Session management is the process of maintaining and controlling user sessions. It involves assigning a unique session identifier/token to each user, which is stored on the server side and used to retrieve relevant session data. The key components of session management are session creation - when a user initiates a session by accessing a web application, a unique session ID is generated for that user; this ID identifies and associates the user’s interactions with the session session tracking - server keeps track of active sessions by associating each session ID with relevant user data session timeout - predefined period to ensure that inactive sessions are automatically terminated session termination - users can manually terminate their sessions by logging out of the application.; when session is terminated, all associated session data is cleared, and the session ID is invalidated session security - measures to protect against session hijacking or fixation attacks We discuss the following alternatives for storing/sending session tokens and compare advantages and weaknesses of these solutions browser cookie URL embedding or passing as GET request parameter hidden HTML form field
